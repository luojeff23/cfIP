# Agent Notes

这个文件不是程序运行必需品。

它只给 AI 编码助手用，作用是让后续会话快速理解这个仓库该怎么改，避免每次从零猜。

## AI 进入仓库后先看什么

1. `docs/01-项目目标与验收.md`
2. `docs/02-架构与开发规则.md`
3. `scripts/check.ps1`

## 当前协作规则

- 先写清本次目标和验收标准，再改代码
- 优先补测试，再改实现
- 改完先跑 `.\scripts\check.ps1`
- `main.go` / `request_logic.go` 放调度和可测试规则
- `scanner/` 放底层网络探测
- `static/` 只放前端展示和交互

## 最小检查清单

- Go 代码保持 `gofmt`
- 影响逻辑的改动，至少更新相关 `_test.go`
- 不要把核心规则塞进前端文件
- 不要在没有测试保护的情况下大改结构
